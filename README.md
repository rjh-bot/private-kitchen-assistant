# 私人厨房助理 (Private Kitchen Assistant)

AI 驱动的个性化饮食管理助手。通过 **LLM（大语言模型）**、**RAG（检索增强生成）**、**Agent 智能体架构**和**多模态识别技术**，为每个用户提供量身定制的饮食规划、营养分析和菜谱推荐服务。

---

## 背景

「吃什么」是每个人每天都要面对的问题。对于有特定健康目标（减脂、增肌、控糖）或饮食限制的人群，日常饮食管理更需要精细化、个性化的指导。然而市面上多数饮食管理工具要么过于通用，缺乏针对个人目标的动态调整能力；要么使用门槛高，交互方式不自然。

**私人厨房助理**通过自然语言对话界面，结合 Agent 智能体推理、RAG 菜谱检索和多模态图片识别，让用户像跟营养师聊天一样管理自己的饮食。系统会根据用户的身体数据、健康目标和饮食偏好，在对话过程中动态调用不同工具（搜索菜谱、分析营养、生成计划），给出真正有价值的反馈。

## 核心功能

### 1. 智能对话饮食管家

系统基于 **ReAct（推理-行动-观察）** 范式构建了一套完整的 Agent 闭环。当用户在对话界面提问时：

1. **意图分析** — Agent 理解用户的真实需求（是想找菜谱、算热量、还是生成计划）
2. **工具调用** — 根据需要自动调用对应的工具函数
3. **结果整合** — 将工具返回的结构化数据与 LLM 的生成能力结合，转化为自然语言回复
4. **SSE 流式输出** — 实时展示 Agent 的推理过程和工具调用结果，用户可以看到"思考中→调用了什么工具→得出什么结论"的完整链路

### 2. 菜谱 RAG 检索

内置菜谱库通过 **混合检索** 实现高效的菜谱查找：

- **向量检索** — 使用 BAAI/bge-small-zh-v1.5 中文嵌入模型，理解语义层面的匹配（如"鸡胸肉怎么做"能匹配到鸡胸肉相关菜谱）
- **BM25 关键词检索** — 基于词频统计的传统检索，保证精确的关键词命中
- **混合评分** — 两种检索结果加权融合（各 50%），兼顾语义理解与精确匹配
- **L1（精确）+ L2（语义近似）双层缓存** — 加速高频查询，避免重复计算

### 3. 营养分析

内置常见食物营养数据库（鸡胸肉、鸡蛋、米饭、牛肉等十余种日常食材），支持：

- 按食物名称查询每 100g 的营养成分（热量、蛋白质、脂肪、碳水化合物）
- 按实际重量换算营养数据
- 对一餐中的多种食物进行总热量汇总计算

### 4. 个性化饮食计划

根据用户的健康目标自动生成饮食计划：

- **4 种目标模板** — 减脂、增肌、控糖、健康饮食，各有不同的配餐逻辑
- 支持自定义每日总热量和目标天数（1/3/7 天）
- 四餐结构：早餐、午餐、晚餐、加餐，每餐热量分配合理
- 每日随机组合，计划丰富不重复

### 5. 多模态图片识别

用户可拍照上传食材或菜品图片，系统通过 DeepSeek 的多模态视觉能力进行识别，返回：

- 图中食材/菜品的名称
- 每类食材的估算营养成分（热量、蛋白质、脂肪、碳水）
- 多种食材时分别列出并给出汇总

### 6. 个人档案与个性化

系统根据用户的个人档案参数提供差异化服务：

| 字段 | 说明 |
|------|------|
| 健康目标 | 减脂 / 增肌 / 控糖 / 健康饮食 |
| 体重 / 身高 | 身体基础数据 |
| 饮食偏好 | 如"喜欢清淡""偏爱鸡肉" |
| 过敏原 / 忌口 | 如"花生过敏""不吃辣" |
| 长期饮食指令 | 如"每餐保证蛋白质，少油少盐" |

这些信息会注入到对话的系统提示词中，使 Agent 的回复始终贴合用户个人情况。

### 7. Subagent 专家体系

除通用的对话 Agent 外，系统还内置了两个垂直领域的专家子代理：

- **营养师** — 专门处理营养评估、热量计算、饮食方案设计等任务，回答更具专业性
- **烹饪顾问** — 专注于菜谱推荐、烹饪技巧指导、食材搭配等实操问题

AgentHub 根据用户问题自动判断是否需要调用 Subagent，实现"一个入口，多位专家"的交互体验。

### 8. 饮食记录

支持用户在对话中记录每餐的进食内容，培养持续追踪饮食的习惯。


## 技术栈

| 层面 | 技术选型 |
|------|----------|
| 后端框架 | Python / FastAPI |
| 前端框架 | React 18 / Vite |
| 大语言模型 | DeepSeek API (deepseek-chat + deepseek-vl2) |
| 向量嵌入 | BAAI/bge-small-zh-v1.5 |
| 关键词检索 | BM25 (rank-bm25) |
| 数据库 | SQLite + SQLAlchemy (async) + aiosqlite |
| 缓存 | 内存级 L1（精确）+ L2（语义近似）双层缓存 |
| Agent 架构 | ReAct + AgentHub + Subagent |
| 流式通信 | Server-Sent Events (SSE) |
